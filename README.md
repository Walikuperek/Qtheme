# Qtheme
* [GitHub repository](https://github.com/Walikuperek/Qtheme)
* [NPM package](https://www.npmjs.com/package/@quak.lib/qtheme)

~8kb.js, simple, fast and production-ready library for managing themes in your app. Create multiple themes and switch between them with **`Qtheme.setTheme(theme)`**

Tested in **Angular**, **React**. Should work with any framework.

> Based on *document.documentElement.style.setProperty* so will work with **vanilla JS** if supports import/export
> 
> Try [vitejs.dev](https://vitejs.dev/) to use powerful import/export in vanilla JS

## Install
```
npm install @quak.lib/qtheme
```

## Getting started
**Qtheme** main goal is to provide easy way to manage themes in your app (JS/TS/any framework). It's not a CSS framework, it's just a library that will help you to manage themes.

But... Qtheme can generate CSS for you(for theme atoms), so you don't have to write it by yourself. It's optional, you can use it or not.

To sum up, Qtheme lets you:
* Create infinite number of themes
* Switch between them easily
* Set common theme atoms for all themes
* Initialize already chosen theme on app start
* Generate CSS classes for you, so you don't have to write it by yourself

### How does it work?

```typescript
import {Qtheme, Theme} from '@quak.lib/qtheme'

const darkTheme: Theme = {
  name: 'dark',
  atoms: [
      // [AtomName, AtomValue]
      ['bg-color', 'background:#000'], // will gen. CSS class .bg-color { background: var(--bg-color) }
      ['text-color', 'color:#fff'], // will gen. CSS class .text-color { color: var(--text-color) }
      ['primary', 'dodgerblue']
  ] 
};
const purpleDarkTheme: Theme = {
  name: 'dark',
  atoms: [
      ['bg-color', 'background:#000'],
      ['text-color', 'color:#fff'],
      ['primary', 'purple']
  ] 
};

// Initialize theme
Qtheme.setTheme(darkTheme)

// Initialize already chosen theme
const savedTheme: Theme | null = Qtheme.getTheme()
if (savedTheme) {
  Qtheme.setTheme(savedTheme)
} 

// Switch theme
Qtheme.setTheme(purpleDarkTheme)
```
Since we did not provide any `background-color:`, `color:` or anything else to `'primary'` value, we need to add in CSS
```css
.primary-color {
  color: var(--primary);
}

/* Or some for primary background */
.bg-primary {
  background-color: var(--primary);
}
```
Then in html
```html
<div class="bg-color">
  <p class="text-color">Hello TS world!</p>
  <button class="bg-primary text-color">Click me!</button>
</div>
```

### Autogenerated CSS classes
> You should use auto-generated CSS classes. It's less code for you ;)

If you want to use auto-generated CSS classes then you need to provide `color:` or `background:` or what you need. Just follow regular CSS syntax.

#### Examples

| **ThemeAtom**                               | **:root variable**                  | **generated CSS**                            |
|---------------------------------------------|-------------------------------------|----------------------------------------------|
| 'primary-color', 'color:hsl(53, 100%, 50%)' | --primary-color: hsl(53, 100%, 50%) | .primary-color {color:var(--primary-color)}  |
| 'bg-color', 'background-color:#fff'         | --bg-color: #fff                    | .bg-color {background-color:var(--bg-color)} |
| 'text-color', '#fff'                        | --text-color: #fff                  | <won't generate CSS>                         |
| ...                                         | ...                                 | ...                                          |

### Declaring Themes
> **Remeber to override atoms in different themes!**

```typescript
import {Qtheme, Theme, ThemeAtom} from '@quak.lib/qtheme'

// Import and and follow this interface
interface Theme {
  name: string;
  atoms: ThemeAtom[];
}

// bgColor and bg-color will be converted to --bg-color
const lightTheme: Theme = {
  name: 'light',
  atoms: [
      ['bgColor', 'background:#fff'], // will be converted to --bg-color
      // will gen. CSS class .bg-color { background: var(--bg-color) }    
          
      ['textColor', 'color:#000'], // will be converted to --text-color
      // will gen. CSS class .text-color { color: var(--text-color) }    
          
      ['primary', 'dodgerblue'] // will be converted to --primary
  ]
};

const darkTheme: Theme = {
  name: 'dark',
  atoms: [
      ['bg-color', 'background:#000'], // will gen. CSS class .bg-color { background: var(--bg-color) }
      ['text-color', 'color:#fff'], // will gen. CSS class .text-color { color: var(--text-color) }
      ['primary', 'dodgerblue']
  ] 
};

// 'bgColor' and 'bg-color'... it will work, but better keep the same atom naming convention
```

> **AtomValue** can be any CSS value like: #fff, rgba(0,0,0,0.5), 16px, 50vw, etc.

And if you want to use CSS variables then you can use them like: `var(--atom-name)`


## Features

### interface SetRootAtomsOptions
You can pass partial options to `setTheme` and `setCommonAtoms` methods
```typescript
import {SetRootAtomsOptions} from '@quak.lib/qtheme'

interface SetRootAtomsOptions {
  generateCSS: boolean; // default true - will generate CSS classes for you if 'color:your-value' or 'background-color:your-value' or what you need is provided
  token: string; // default 'Qtheme' - localStorage key
  commonToken: string; // default 'Qtheme-common' - localStorage key for common atoms
}

const options = {
    generateCSS: false, // default true
};
Qtheme.setTheme(theme, options);
Qtheme.setCommonAtoms(atoms, options);
```

### interface Qtheme
```typescript
// Set theme - light, dark, purpleDark, highContrast, etc.
Qtheme.setTheme(theme: Theme, options?: Partial<SetRootAtomsOptions>): void
```

```typescript
// To init already chosen theme: first get, then set
Qtheme.getTheme(token = 'Qtheme'): Theme | null
```

```typescript
// Set common atoms - atoms that will be applied to all themes
Qtheme.setCommonAtoms(atoms: ThemeAtom[], options?: Partial<SetRootAtomsOptions>): void
```

```typescript
// For init already chosen ones, etc.
Qtheme.getCommonAtoms(commonAtomsToken = 'Qtheme-common'): ThemeAtom[]
```


## Tips

### Disabling Generating CSS classes
```typescript
Qtheme.setTheme(yourTheme, { generateCSS: false })
Qtheme.setCommonAtoms(yourCommonAtoms, { generateCSS: false })
```

### Changing localStorage keys
Change theme localStorage key - default(*'Qtheme'*)
```typescript
Qtheme.setTheme(yourTheme, { token: 'YourNewLocalStorageThemeKey' })

const currentTheme = JSON.parse(localStorage.getItem('YourNewLocalStorageThemeKey'))
const equivalent = Qtheme.getTheme('YourNewLocalStorageThemeKey')
```
Change common atoms localStorage key - default(*'Qtheme-common'*)
```typescript
Qtheme.setCommonAtoms(yourTheme, { commonToken: 'YourNewLocalStorageCommonAtomsKeys' })

const currentCommonThemeAtoms = JSON.parse(localStorage.getItem('YourNewLocalStorageCommonAtomsKeys'))
const equivalent = Qtheme.getCommonAtoms('YourNewLocalStorageCommonAtomsKeys')
```

### :root variables
You can add CSS :root with default values, actually it's optional, but it's good to have it.
```css
/* You can still use :root, no worries, just keep unique names */
:root {
    --bg-color: #fff;
    --text-color: #000;
    --primary: dodgerblue;
}
```

### Set common atoms
If you want to set common atoms for all themes, you can use `setCommonAtoms` method
```typescript
import {Qtheme, Theme, ThemeAtom} from '@quak.lib/qtheme';

// Use unique names for every atom in your app, otherwise it will be overwritten
const commonAtoms: ThemeAtom[] = [
  ['font-family', 'sans-serif'],
  ['font-size', 'font-size:16px'] // will provide CSS for -> class="font-size" with font-size:16px
];

Qtheme.setCommonAtoms(commonAtoms);

// Get common atoms, you can save it to DB, etc.
const atoms = Qtheme.getCommonAtoms();
```

## Angular usage
In *app.component.ts*
```typescript
import {Qtheme} from '@quak.lib/qtheme';
import {lightTheme, darkTheme} from '../themes'; // import your themes

// regular angular imports and @Component...
class AppComponent {
  constructor() {
    Qtheme.setTheme(darkTheme);
  }
  
  setLightTheme(): void {
    Qtheme.setTheme(lightTheme);
  }
}
```
Then in *app.component.html*
```html
<div class="bg-color">
  <button (click)="setLightTheme()">Set light theme</button>
  <p class="text-color">Hello Angular world!</p>
</div>
```

## React usage
Take a look how it's done with Next.js, it's the same with Gatsby, etc.

### Example with Next.js
In */pages/index.tsx*

```tsx
import {useEffect} from 'react';
import {Qtheme} from '@quak.lib/qtheme';
import {lightTheme, darkTheme} from '../themes'; // import your themes

export function Index() {
  // Simply   
  useEffect(() => {
    Qtheme.setTheme(lightTheme);
  });

  setTimeout(() => { // use setTimeout if you want
    if (typeof window !== 'undefined') {
      Qtheme.setTheme(lightTheme);
    }
  });

  return (
    <div className="bg-color">
      <button onClick={() => Qtheme.setTheme(darkTheme)}>Set dark theme</button>
      <p className="text-color">Hello React world!</p>
    </div>
  );
}
```
